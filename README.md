# Practice Test

Static quiz app for Microsoft AI agentic-architecture practice questions. All answers are grounded in Microsoft Learn documentation with source URLs included.

Live site: enable GitHub Pages on this repo (Settings → Pages → Deploy from a branch, `master`, `/ (root)`), then open `https://<user>.github.io/practicetest/app/`.

## What's in here

```
practicetest/
├── build.py             # parses extracted/*.md → app/questions.json
├── README.md            # this file
├── CLAUDE.md            # project context for Claude Code

└── app/

    ├── style.css
    ├── app.js
    └── questions.json   # generated by build.py
```

103 questions total. 16 reference a case study (8 Fabrikam, 8 Contoso). Each question has a matching `_answer.md` next to it.

## Use the app

### Local

```
cd app
py -m http.server 8000
```

Open http://localhost:8000. Opening `index.html` directly via `file://` will not work (the page uses `fetch()`).

### Behavior

- Randomizes question order per session, no repeats until all are seen.
- Progress persists in `sessionStorage` (a refresh resumes where you were; closing the tab clears it).
- Multiple-choice with one correct letter: click an option to commit, instant right/wrong feedback.
- Hotspot, drag-drop, multi-select, yes-no series: a "Reveal Answer" button shows the correct answer and explanation.
- Case-study questions show a `Case Study: Fabrikam` or `Case Study: Contoso` tag.
- Each reveal shows the full grounded explanation rendered from markdown, including clickable Microsoft Learn URLs.
- Keyboard: `Space` to reveal, `→` for next.

## Rebuild

After editing or adding any question or answer file, regenerate the JSON:

```
py build.py
```

This walks `extracted/topic-{1,2,3}/`, parses each `qNN.md` + `qNN_answer.md`, and writes `app/questions.json`.

## Deploy on GitHub Pages

1. Push to GitHub (already done if this came from `gh repo create`).
2. Repo → Settings → Pages → Source: **Deploy from a branch**, branch `master`, folder `/ (root)`, Save.
3. Wait ~1 minute for the first build.
4. Open `https://<user>.github.io/practicetest/app/`.
