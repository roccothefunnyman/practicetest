# Practice Test

Static quiz app for Microsoft AB-100 (Agentic AI Business Solutions Architect) practice questions. Every answer is grounded in Microsoft Learn with source URLs included.

Live site: enable GitHub Pages on this repo (Settings → Pages → Deploy from a branch, `master`, `/ (root)`), then open `https://<user>.github.io/practicetest/app/`.

## Use the app

### Local

```
cd app
py -m http.server 8000
```

Open http://localhost:8000. Opening `index.html` directly via `file://` will not work (the page uses `fetch()`).

### Behavior

- **New session picker.** Click `New session` in the topbar (or `Start New Session` on the summary screen) to open the picker modal. Quick-pick chips (All / None / Wrong only / Unanswered / Correct only / Plan only / Design only / Deploy only / Fabrikam case / Contoso case / No case study), per-domain All/None links, and per-question checkboxes. The session that follows includes only the selected questions, and every progress bar reflects that subset.
- **Per-domain progress bars.** Top bar shows overall correct/wrong; below it, three bars for AB-100 domains (Plan 25–30%, Design 25–30%, Deploy 40–45%). Mastery (correct/wrong per question) persists in `localStorage` across sessions. `Reset progress` in the topbar wipes it.
- **Light / dark theme.** Toggle in the topbar; the case-study visuals share the same setting via `localStorage.quiz_theme`.
- **Case-study popup.** Any case-study question shows a `View case study` button that opens the matching Fabrikam or Contoso visual in a modal iframe. Press `C` to open it; `Esc` or backdrop click to close.
- **Slide controls.** Multiple-choice commits on click. Multi-select / hotspot / drag-drop use a `Submit Answer` button. `Back` returns to the previous question (also `←`). `Clear` wipes the current selections so you can retry. `Copy Question` copies the stem and lettered options to the clipboard for multiple-choice and multi-select.
- **Keyboard:** `Space` reveal, `→` next, `←` back, `C` open case study, `Esc` close any modal.
- **Session order** is randomized; in-progress state persists in `sessionStorage` (a refresh resumes where you were; closing the tab clears it). Mastery stats persist in `localStorage` across sessions.

## Rebuild

After editing or adding any question or answer file, regenerate the JSON:

```
py build.py
```

This walks `extracted/topic-{1,2,3}/`, parses each `qNN.md` + `qNN_answer.md`, merges in `app/categories.json`, and writes `app/questions.json` along with a domain-distribution summary.

## Cache busting

`index.html` references `style.css?v=N` and `app.js?v=N`. Bump `N` whenever you ship breaking changes to the JS or CSS so browsers fetch fresh files instead of running stale cached code against new markup.

## Deploy on GitHub Pages

1. Push to GitHub.
2. Repo → Settings → Pages → Source: **Deploy from a branch**, branch `master`, folder `/ (root)`, Save.
3. Wait ~1 minute for the first build.
4. Open `https://<user>.github.io/practicetest/app/`.

Pages serves the whole repo from the root, so the in-app case-study modal can iframe `../extracted/topic-1/case-study.html` directly.
