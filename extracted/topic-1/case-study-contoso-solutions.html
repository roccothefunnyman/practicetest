Foundation: AI adoption & governance Cross-cutting

  • Use the Cloud Adoption Framework for AI (Strategy → Plan → Ready → Govern → Secure → Manage) and the AI agent adoption sub-track for the agent-specific decisions.
  • Multiple legal entities and Azure subscriptions: lay down Azure landing zones for the platform, and one Power Platform environment per legal-entity / data-residency boundary so DLP, Dataverse, and Copilot Studio agents are scoped per entity.
  • Every initiative must have a business case, defined audience profile, and ROI — this is the gate the CFO and CEO use.
  • Anchor responsible AI in the six Microsoft principles: fairness, reliability and safety, privacy and security, inclusiveness, transparency, accountability.

R&D MCP server reuse Existing technology

  • The case study explicitly states R&D already runs an MCP server with product specs and compliance data.
  • Plug it straight into Copilot Studio with the MCP onboarding wizard. No new connector code required.
  • This is also why the agent must use generative orchestration — MCP integration in Copilot Studio is generative-only.
  • Patents and engineering designs are sensitive: apply Purview labels to the underlying Dataverse columns and let the labels propagate through the agent's responses.

Why each pick beats the alternative

  • Supplier Communications Agent over Release Planner / Trace Parser. The brief is a manufacturing SCM use case with rapid-growth ROI. Procurement automation is the highest-leverage prebuilt option.
  • Generative orchestration over classic. The case study calls out "topics selected by description, not trigger phrase" — that is the literal definition of generative orchestration.
  • MCP for the existing R&D server over a custom Power Platform connector. The R&D MCP server already exists. MCP is the supported, no-code path to surface it in Copilot Studio.
  • A2A for "connect to other agents based on description" over HTTP connectors. A2A was built for cross-platform agent-to-agent communication with capability discovery via agent cards.
  • Defender XDR AI agent inventory over a custom CMDB entry. The inventory natively discovers Copilot Studio and Foundry agents and surfaces posture insights without manual upkeep.
  • Purview DSPM for AI Activity Explorer over Application Insights. Activity Explorer classifies sensitive info types automatically and ties them to the user; Application Insights would require building this from scratch.
  • Foundry safety evaluations + Responsible AI dashboard over a generic governance review. They are the Microsoft-recommended tools that map directly to the four metrics (reliability, interpretability, fairness, compliance) the CEO requires.
