Fabrikam, Inc.

Topic 1 case study (also referenced in Topic 2 and Topic 3).
Global consumer goods B2B sales team On-prem today Cloud + AI-first target First AI agent for the company

Current state

  • On-prem proprietary sales tools
  • Corporate PCs with physical keyboards required
  • Disparate systems and file servers
  • Reps spend time searching, not engaging
  • No mobile access during customer visits
→

Target state

  • Full Microsoft cloud migration
  • Dynamics 365 Sales replaces proprietary CRM
  • Dataverse as the single source of truth
  • One AI agent, low-code, hands-free voice
  • Sales managers can intervene on stale threads

Pain points Why AI

  • Keyboard-only systems block mobile use during customer interactions.
  • Sales executives lose time hunting across systems instead of focusing on customers.
  • Stakeholders fear low AI adoption, which would block ROI and future AI funding.
  • This is Fabrikam's first AI agent, so rapid adoption is critical.

Infrastructure migration Initiative

  • Migrate everything from on-prem to Microsoft cloud.
  • Includes user authentication and the security framework.
  • Consolidate all data into a Dataverse environment as the SSOT.
  • Adoption of services by end users is in scope, not just lift and shift.

Sales cycle enablement Initiative

  • Build one AI agent using low-code, with Dataverse as its core.
  • Sales managers can see unanswered prospect correspondence and step in.
  • Replace proprietary CRM with Dynamics 365 Sales.
  • Sales execs use Dynamics 365 Sales to track opportunities and follow ups.
  • Hands-free headsets to talk to the AI agent about policy and customer data.

Infrastructure migration requirements Must

  • Azure for all future infrastructure workloads.
  • Follow Microsoft-recommended cloud migration methodology (Cloud Adoption Framework).
  • Every AI agent must have a calculated ROI that proves savings.

Sales cycle enablement requirements Must

  • AI agent must follow Microsoft conversational UX recommendations.
  • A designated checklist must be reviewed for compliant deployment.
  • Detailed telemetry on the first agent for troubleshooting and tuning during launch.
  • Unexpected agent actions must escalate to a live rep (example: two failed attempts).
  • ROI of new vs. old process is required for stakeholder sign-off.
  • Dynamics 365 Sales must speed up correspondence with prospects.
  • Sales managers report agent adoption to stakeholders monthly.
  • Any sensitive info (user IDs, names) shared via the agent must be tracked for audit.

Likely Microsoft tech in play

Azure all future workloads Cloud Adoption Framework migration methodology Microsoft Dataverse single source of truth Dynamics 365 Sales CRM replacement Microsoft Copilot Studio low-code AI agent Conversational UX guidance agent design Copilot Studio deployment checklist compliant launch Agent telemetry / analytics monitoring Microsoft Purview sensitive data tracking and audit Live agent escalation fallback path

Quick recall hooks

  • One agent, low-code, Dataverse-grounded = Copilot Studio.
  • Microsoft-recommended migration methodology = Cloud Adoption Framework.
  • Hands-free voice interaction implies conversational UX guidelines and voice channel design.
  • "Track sensitive info for audit" typically points to Purview / DLP / agent audit logs.
  • Adoption is the funding gate. Any choice that increases friction is wrong.
