Two AI agents to deploy Initial phase

Prebuilt agent in D365 SCM

  • CTO has not picked one yet.
  • CTO wants to view available agent templates to find max business value.
  • Agent capabilities must be previewed in a discovery meeting with relevant stakeholders.

Custom Copilot Studio agent

  • Built as a low-code solution.
  • Uses data from D365 Supply Chain Management.
  • Answers questions for the manufacturing team.
  • Accessible from Microsoft Teams.
  • Accessible from inside D365 Supply Chain Management.
  • Must eventually connect to other agents, selected by description (multi-agent orchestration).
  • Topics selected by description, not trigger phrase (generative orchestration).
  • Must answer product spec questions using existing tech (the R&D MCP server).
  • Must use D365 SCM business logic stored outside the application (Dataverse plug-ins / external code).

AI adoption guardrails Across the board

  • Every initiative needs a business use case.
  • Every initiative needs a defined audience profile.
  • Every initiative needs estimated ROI comparing current cost to AI cost.
  • R&D's existing MCP server is the source for product specs and compliance, do not duplicate it.

Analysis, reporting, troubleshooting Ongoing

  • Monthly CISO audit for compliance and security.
  • Quarterly CFO ROI review using the Copilot Studio agent usage estimator.
  • Track sensitive data access per agent run (user + volume).
  • Capture user feedback on response quality during interactions.
  • Quarterly CEO assessment, Microsoft-recommended tools, must cover reliability, interpretability, fairness, compliance.
  • Daily abandoned vs. resolved session count, watch for inefficient credit usage.
