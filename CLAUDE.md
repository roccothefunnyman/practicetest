Context for Claude Code when working in this repo.

## What this project is

A static quiz app for Microsoft AI agentic-architecture certification practice questions. The questions came from an ExamTopics PDF (kept locally, gitignored), were extracted into per-question markdown files under `extracted/`, and each question has a matching `_answer.md` file containing the correct answer and an explanation grounded in Microsoft Learn documentation.

The app at `app/` is plain HTML / CSS / vanilla JS. It loads `app/questions.json` (generated by `build.py`), shuffles the questions, and presents them slide by slide with reveal-style explanations.

The repo is published as a public GitHub Pages site so Rocco can study from any device.

## Question file format

`extracted/topic-N/qNN.md`:

```markdown
---
topic: 1
question: 2
type: multiple-choice
case_study: true
---

# Question 2
> See [case-study.md](case-study.md)

Which framework should you use for the infrastructure migration?

A. Microsoft Cloud Adoption Framework for Azure
B. Success by Design
C. Microsoft Power Platform Center of Excellence (CoE)
D. Microsoft Power Platform Project Setup Wizard
```

Frontmatter fields:
- `topic`: 1, 2, or 3
- `question`: integer matching the file name
- `type`: `multiple-choice`, `multi-select`, `hotspot`, `drag-drop`, or `yes-no-series`
- `case_study`: `true` if the question references a case study, otherwise `false`

The body rules:
- First non-frontmatter line is `# Question N`
- A blockquote `> See [case-study.md](case-study.md)` indicates which case study file to read. Topic 1 has two distinct files (`case-study.md` = Fabrikam, `case-study-contoso.md` = Contoso). Topic 3 has both companies in one file (questions 1-3 = Fabrikam, questions 39-42 = Contoso).
- The question stem follows.
- For `multiple-choice` and `multi-select`, options appear as `A. ...`, `B. ...`, etc.
- Other types use free-form description (hotspot dropdowns, drag targets, yes/no statements).

## Answer file format

`extracted/topic-N/qNN_answer.md`:

```markdown
---
topic: 1
question: 2
type: multiple-choice
case_study: true
answer: A
---

# Question 2 — Answer

**Correct answer: A. Microsoft Cloud Adoption Framework for Azure**

## Why A is correct

(explanation grounded in case study and Microsoft Learn docs)

## Why the other options are wrong

**B. Success by Design** is for Dynamics 365 implementations, not infrastructure migration...

## Microsoft Learn references

- What is the Microsoft Cloud Adoption Framework? — https://learn.microsoft.com/azure/cloud-adoption-framework/overview
- Migrate workloads to Azure — https://learn.microsoft.com/azure/migration/migrate-to-azure
```

The `answer` frontmatter is the short answer string (a letter for single-choice, a list / description for other types). The body sections are markdown rendered to HTML in the app via marked.js.

## Build

Re-run after any question or answer file changes:

```
py build.py
```

Writes `app/questions.json`. The build script (`build.py`) is self-contained, no Python dependencies beyond the stdlib. It detects case-study membership using the link target in the question body, and for topic 3 falls back to the question number range. If neither signals a company name, it scans the question + answer text for the known names (Fabrikam, Contoso, Adatum, Litware, Northwind, Tailwind).

## Run locally

```
cd app
py -m http.server 8000
```

Then http://localhost:8000. Opening `index.html` directly via `file://` will not work because the page uses `fetch()` to load `questions.json`.

## Deploy

GitHub Pages, public repo:

1. Settings → Pages → Source: Deploy from a branch, `master`, folder `/ (root)`, Save.
2. URL: `https://<user>.github.io/practicetest/app/`.

The app fetches `questions.json` via a relative path, so it works under any subpath.

## When adding new questions

1. Drop the new `qNN.md` file in the right `extracted/topic-N/` folder. Match the existing frontmatter shape.
2. Write the matching `qNN_answer.md`. Ground every answer in Microsoft Learn (use the Microsoft Learn MCP server's `microsoft_docs_search` and `microsoft_docs_fetch` tools). Every answer must include at least one `learn.microsoft.com` URL in the references section.
3. Run `py build.py` to refresh `app/questions.json`.
4. Commit and push. GitHub Pages redeploys automatically.

For bulk additions, spawn one sub-agent per topic in parallel (the original 103 answers were generated this way).

## Writing style for new answers

These rules came from Rocco's main `my.ai/CLAUDE.md` and were applied to every existing answer file. Keep applying them:

- Direct, declarative, professional voice. No hype, no buzzwords.
- No em dashes. Use commas, parentheses, or colons instead.
- No "not X, but Y" / "it's not X, it's Y" / mirrored contrast framing.
- Banned phrases: "bottom line", "why it matters", "sits at the intersection of", "intersect/intersection" used to describe role/skill combinations.
- No trailing "rather than X" / "instead of X" rhetorical clauses at sentence end.
- Plain English, concrete wording. Acknowledge real uncertainty.
- Microsoft technology content must be grounded in Microsoft Learn. Do not fabricate service names, features, or configuration.
- Only use emojis if explicitly requested.

## Case study handling in the app

The app does not display the case study text inline. It only shows a `Case Study: Fabrikam` or `Case Study: Contoso` tag on the question slide. Rocco's call: he wants to test recall of the question logic, not be re-prompted with the full case study every time.

If you want to display case studies inline later, the data path already exists: extend `build.py` to read each `case-study*.md`, attach to the question record, and render in `app.js` behind a "Show case study" toggle.
