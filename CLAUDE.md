## What this project is

A static quiz app for Microsoft **AB-100** (Agentic AI Business Solutions Architect) practice questions. The questions came from an ExamTopics PDF (kept locally, gitignored), were extracted into per-question markdown files under `extracted/`, and each question has a matching `_answer.md` file containing the correct answer and an explanation grounded in Microsoft Learn documentation.

The app at `app/` is plain HTML / CSS / vanilla JS. It loads `app/questions.json` (generated by `build.py`), shuffles the user-selected subset, and presents questions slide by slide with reveal-style explanations.

The repo is published as a public GitHub Pages site so Rocco can study from any device.

## AB-100 domains

The official skills outline (https://learn.microsoft.com/credentials/certifications/resources/study-guides/ab-100) groups the exam into three domains. Every question is tagged with one of these in `app/categories.json` so the app can render per-domain progress bars.

- **Plan AI-powered business solutions** (25–30%) — code: `plan`
- **Design AI-powered business solutions** (25–30%) — code: `design`
- **Deploy AI-powered business solutions** (40–45%) — code: `deploy`

Current bank distribution (`py build.py` prints this on every build): plan 20 (19.4%), design 37 (35.9%), deploy 46 (44.7%). Plan is light vs. the published target; if you re-classify any question, edit `app/categories.json` and rebuild.

## Question file format

`extracted/topic-N/qNN.md`:

```markdown
---
topic: 1
question: 2
type: multiple-choice
case_study: true
---

# Question 2
> See [case-study.md](case-study.md)

Which framework should you use for the infrastructure migration?

A. Microsoft Cloud Adoption Framework for Azure
B. Success by Design
C. Microsoft Power Platform Center of Excellence (CoE)
D. Microsoft Power Platform Project Setup Wizard
```

Frontmatter fields:
- `topic`: 1, 2, or 3
- `question`: integer matching the file name
- `type`: `multiple-choice`, `multi-select`, `hotspot`, `drag-drop`, or `yes-no-series`
- `case_study`: `true` if the question references a case study, otherwise `false`

The body rules:
- First non-frontmatter line is `# Question N`
- A blockquote `> See [case-study.md](case-study.md)` indicates which case study file to read. Topic 1 has two distinct files (`case-study.md` = Fabrikam, `case-study-contoso.md` = Contoso). Topic 3 has both companies in one file (questions 1-3 = Fabrikam, questions 39-42 = Contoso).
- The question stem follows.
- For `multiple-choice` and `multi-select`, options appear as `A. ...`, `B. ...`, etc.
- Other types use free-form description (hotspot dropdowns, drag targets, yes/no statements).

## Answer file format

`extracted/topic-N/qNN_answer.md`:

```markdown
---
topic: 1
question: 2
type: multiple-choice
case_study: true
answer: A
---

# Question 2 — Answer

**Correct answer: A. Microsoft Cloud Adoption Framework for Azure**

## Why A is correct

(explanation grounded in case study and Microsoft Learn docs)

## Why the other options are wrong

**B. Success by Design** is for Dynamics 365 implementations, not infrastructure migration...

## Microsoft Learn references

- What is the Microsoft Cloud Adoption Framework? — https://learn.microsoft.com/azure/cloud-adoption-framework/overview
- Migrate workloads to Azure — https://learn.microsoft.com/azure/migration/migrate-to-azure
```

The `answer` frontmatter is the short answer string (a letter for single-choice, a list / description for other types). The body sections are markdown rendered to HTML in the app via marked.js.

## Build

Re-run after any question, answer, or category change:

```
py build.py
```

Writes `app/questions.json`. The build script:
- Walks `extracted/topic-{1,2,3}/` and parses each `qNN.md` + `qNN_answer.md`.
- Reads `app/categories.json` and stamps each question record with its `category` (`plan` / `design` / `deploy`).
- Detects case-study membership using the link target in the question body, and for topic 3 falls back to the question number range. If neither signals a company name, it scans the question + answer text for the known names (Fabrikam, Contoso, Adatum, Litware, Northwind, Tailwind).
- Emits the `domains` lookup the front end uses for the per-domain progress bars.
- Self-contained, no Python dependencies beyond the stdlib.
- Prints the AB-100 domain distribution and any uncategorized question ids on every build.

## Run locally

```
cd app
py -m http.server 8000
```

Then http://localhost:8000. Opening `index.html` directly via `file://` will not work because the page uses `fetch()` to load `questions.json`.

## Deploy

GitHub Pages, public repo, branch `master`, root folder. URL: `https://<user>.github.io/practicetest/app/`. Pages serves the entire repo from the root, so `app/` can iframe `../extracted/topic-1/case-study.html` to show the case-study visual.

### Cache busting

`index.html` loads `style.css?v=N` and `app.js?v=N`. **Bump N whenever the JS or CSS changes in a way that depends on new markup IDs.** Otherwise browsers will load the new HTML against the cached old JS, hit `null` on a renamed/deleted element, and the page silently breaks (already happened once during the picker rollout). Increment in lockstep across both files.

## App features (so you don't reinvent)

- **New session picker (modal).** Topbar `New session` button opens the picker. Quick-pick chips: All / None / Wrong only / Unanswered / Correct only / Plan only / Design only / Deploy only / Fabrikam case / Contoso case / No case study. Per-domain All/None links. Per-question checkboxes with a one-line stem preview, type tag, mastery dot, and case tag. Picker is a fixed-position modal with backdrop click and `Esc` to close.
- **Progress bars driven by the active subset.** Top bar `correct / wrong / total` plus three per-domain bars below it. Counts come from the active session order (so picking 1 question shows `0 / 1`, not `0 / 103`).
- **Mastery persistence.** Per-question correct/wrong stored in `localStorage.quiz_stats_v1`, keyed by question id. `Reset progress` in the topbar wipes it. Stats are *latest result wins* (a re-attempt overwrites the prior verdict).
- **Light / dark theme.** Toggle in the topbar; persisted to `localStorage.quiz_theme`. Case-study visuals share that key, and the app passes `?theme=dark|light` when opening the iframe so it loads in the matching mode.
- **Case-study popup.** Topic-1 visuals (`extracted/topic-1/case-study.html`, `case-study-contoso.html`) are self-contained dark/light HTML pages. Any case-study question shows a `View case study` pill button that opens the matching visual in a modal iframe. `C` opens, `Esc` closes. Topic-2 and Topic-3 don't have visuals yet; the app currently routes any Fabrikam/Contoso case-study question to the topic-1 visual regardless of topic.
- **Slide controls.** `Back` (also `←`) goes to the previous question; `Clear` wipes current picks so you can retry; `Copy Question` copies stem + lettered options to the clipboard (multiple-choice and multi-select only — hidden for hotspot and drag-drop). `Reveal Answer` / `Submit Answer` for non-multiple-choice. `Next Question` / `Finish` after reveal.
- **Keyboard:** `Space` reveal, `→` next, `←` back, `C` case study, `Esc` close any modal.
- **Session state.** `sessionStorage.practice_session_v1` holds `{ order, cursor }` for the active session — refresh to resume, close tab to lose. Mastery (`quiz_stats_v1`) and theme (`quiz_theme`) live in `localStorage` so they survive across sessions and tabs.
- **First launch** (no saved session) auto-starts a session with all 103 questions. The picker is only shown when the user clicks `New session` or `Start New Session` from the summary.

## When adding new questions

1. Drop the new `qNN.md` file in the right `extracted/topic-N/` folder. Match the existing frontmatter shape.
2. Write the matching `qNN_answer.md`. Ground every answer in Microsoft Learn (use the Microsoft Learn MCP server's `microsoft_docs_search` and `microsoft_docs_fetch` tools). Every answer must include at least one `learn.microsoft.com` URL in the references section.
3. Add the new question id to `app/categories.json` with its AB-100 domain (`plan` / `design` / `deploy`). `build.py` will print "uncategorized" warnings for any question missing from this map.
4. Run `py build.py` to refresh `app/questions.json`.
5. Commit and push. GitHub Pages redeploys automatically. If you also touched `app.js` or `style.css`, bump the `?v=N` cache-buster on both.

For bulk additions, spawn one sub-agent per topic in parallel (the original 103 answers were generated this way).

## Writing style for new answers

These rules came from Rocco's main `my.ai/CLAUDE.md` and were applied to every existing answer file. Keep applying them:

- Direct, declarative, professional voice. No hype, no buzzwords.
- No em dashes. Use commas, parentheses, or colons instead.
- No "not X, but Y" / "it's not X, it's Y" / mirrored contrast framing.
- Banned phrases: "bottom line", "why it matters", "sits at the intersection of", "intersect/intersection" used to describe role/skill combinations.
- No trailing "rather than X" / "instead of X" rhetorical clauses at sentence end.
- Plain English, concrete wording. Acknowledge real uncertainty.
- Microsoft technology content must be grounded in Microsoft Learn. Do not fabricate service names, features, or configuration.
- Only use emojis if explicitly requested.

## Case study handling in the app

- The slide does *not* inline the case study text. It tags the question with `Case Study: Fabrikam` or `Case Study: Contoso` and shows a `View case study` button that opens the topic-1 visual in a modal iframe.
- Rocco's call: he wants to test recall of the question logic and reach for the case study only when needed.
- Topic-2 and Topic-3 don't have their own visuals yet. The current modal always routes to the topic-1 file based on company name. If the case-study text differs across topics in a way that matters for the question, build a topic-specific visual and update the routing in `app.js`.

## Local dev tips

- The repo is OneDrive-synced. Avoid creating massive build artifacts.
- The `_work/` folder (gitignored) contains the original PDF extraction pipeline. The PDFs themselves are also gitignored. Both stay local.
- `questions.json` is committed (it is a generated artifact, but committing it means GitHub Pages can serve the app without a build step).
- After a JS or CSS change, **bump `?v=N` in `index.html`** before pushing. The picker rollout broke once because new markup loaded against cached old JS — root cause was a missing cache-buster bump.
